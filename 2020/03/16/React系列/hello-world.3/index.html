小程序推出web-view属性后，可以像客户端一样，直接在小程序中嵌入H5页面，极大方便了小程序开发，同时解决了小程序打包后体积较大问题，也让那些诟病小程序太封闭的同学看到希望

web-view使用
web-view组件是一个可以用来承载网页的容器，会自动铺满整个小程序页面；

主要属性：src：网站的url，指向网页的链接。需登录小程序管理后台配置域名白名单。

bindmessage：网页向小程序 postMessage 时，会在特定时机（小程序后退、组件销毁、分享）触发并收到消息。e.detail = { data }


通信
1.H5通知native

这个比较常见，通过jssdk提供的api向主要用于分享传参，页面跳转等。

举个例子，在页面需要分享的时候，一般我们会给native传递分享的标题，链接等

H5端：

wx.miniProgram.postMessage({
data: {
title: ‘分享标题’,
url: ‘xxxx’
}
});
native端：

Page({
bindGetMsg: funtion(event){
// event.detail
// do something
}
)
需要注意的是，bindmessge的回调函数不是实时触发的，该方法只在小程序后退、组件销毁、分享时触发。
